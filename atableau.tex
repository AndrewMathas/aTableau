  \section{Introduction}

  The commands are designed to be easy to use
  and easy to read.  The literature contains many embellishments of the
  basic diagrams, so we provide mechanisms to customise and style these
  diagrams.

  The package provides basic commands for drawing these objects,
  together with styling options and a key-value interface for adding
  extra features. Under the hood, everything is drawn using \TikZ.  It
  is possible to use all of our commands either as standalone commands,
  or as components of a \keyword{tikzpicture} environment.  This makes
  it possible to use these commands inside more complicated diagrams.

  \keyword{\usepackage[options]{atableau]}}

  \section{Young diagrams and tableaux}

    \emph{Partitions}, which are weakly decreasing sequences of
    non-negative integers, are fundamental objects in algebraic
    combinatorics and representation theory.\index{partition}  Rather
    than working with sequences of integers, it is often useful to
    visualise partitions as Young \emph{diagrams}, which are arrays of
    \emph{boxes}, or \emph{nodes}, in the plane. A \emph{tableau} is a (Young) diagram where the
    boxes are labelled using some alphabet. Equivalently, a diagram is a
    tableau with empty labels.
    \index{young diagram@Young diagram|see{diagram}}
    \index{diagram}
    \index{tableau}
    \index{box}
    \index{node}
    \index{node|see{box}}
    \index{exponential notation}
    \index{partition!exponential notation}

    For example, $\lambda=(4,3,3,2,0,0,\dots)$ is a partition of $12$,
    since the entries sum to~$12$. It is customary to omit the zeros
    when writing partitions and to use exponents for repeated parts, so
    we also write partitions in \emph{exponential notation} as
    $\lambda=(4,3^2,2)$. Many authors identity a
    partition~$\lambda=(\lambda_1,\lambda_2,\dots)$ with its
    \emph{diagram}, which is the set
    \[
    [\lambda] = \set{(r,c)|1\le c\le\lambda_r\text{ for }r\ge1}.
    \]
    The diagram of~$\lambda$ is visualised as a justified array
    of boxes in the plane, where the first row has~$4$ boxes, the nest
    two rows have~$3$ boxes and the last row has~$2$ boxes. The
    \emph{shape} of a diagram is the partition that gives the number of
    boxes in each row. Here are some examples of the diagrams and
    tableaux of shape $(2,1^2)$ that can be drawn using \aTableau:

    \begin{center}
      \tikzset{
         B/.style={fill=blue!20},
         G/.style={fill=ForestGreen!20},
         R/.style={fill=red!40,text=white},
         Y/.style={fill=yellow!20}
      }
      \Diagram{2,1^2}\qquad
      \Tableau{[R]1[B]2,[G]3,[Y]4} \qquad
      \Tableau[french]{[R]1[B]2,[G]3,[Y]4} \qquad
      \Tableau[ukrainian]{[R]1[B]2,[G]3,[Y]4} \qquad
      \Tableau[australian]{[R]1[B]2,[G]3,[Y]4}
    \end{center}
    This choice of colouring is gratuitous, but it emphasises that
    \aTableau allows you to add style (and, in particular,
    colour), to every box in these diagrams. This example also shows that
    \aTableau supports the four different conventions used for drawing Young
    diagrams and tableaux. The following table shows how to draw the
    Young diagram of shape $\lambda=(4,3^2,2)$ using the four tableaux
    conventions supported by \aTableau:
    \begin{center}
        \aTabset{align=centre}
        \begin{tabular}{lp{0.6\textwidth}l} \toprule
            Convention & \aTableau command & Diagram \\ \midrule
            English    & \keyword{\Tableau[english]{1234,567,89{10},{11}{12}}}\newline(the default)  & \Tableau{1234,567,89{10},{11}{12}}\\[10mm]
            French     & \keyword{\Tableau[french]{1234,567,89{10},{11}{12}}}     & \Tableau[french]{1234,567,89{10},{11}{12}}\\[10mm]
            Ukrainian\footnotemark
                       & \keyword{\Tableau[ukrainian]{1234,567,89{10},{11}{12}}}  & \Tableau[ukrainian]{1234,567,89{10},{11}{12}}\\[10mm]
            Australian\footnotemark
                       & \keyword{\Tableau[australian]{1234,567,89{10},{11}{12}}} & \Tableau[australian]{1234,567,89{10},{11}{12}} \\[10mm]
            \bottomrule
        \end{tabular}
        \index{english@English}
        \index{french@French}
        \index{ukrainian@Ukrainian}
        \index{australian@Australian}
        \footnotetext[1]{Some authors call this the Russian convention}
        \footnotetext[2]{I have not seen this convention in the literature, but this name for the convention seems apt.}
    \end{center}
    The different conventions for diagrams, and tableaux, are listed in
    order of (my perception of) their popularity in the literature.  By
    default, the \textit{English} convention is used, so
    \keyword{\Tableau{1234,567,89{10},{11}{12}}} produces the first
    tableau above.  These four conventions are used in the same
    way with the \keyword{\Diagram} command, which produces Young
    diagrams. You can change the default convention using the
    \keyword{\aTabset} command.

    This table shows how the commands provided by \aTableau work: there
    are basic commands for drawing diagrams, and these diagrams can be
    embellished using optional arguments and a key-value interface. In
    addition, as the following example shows, by supplying (optional)
    Cartesian coordinates, these diagrams can be used inside a
    \keyword{tikzpicture} environment to create more exotic pictures:

    \begin{Example}
      % \usetikzlibrary{patterns}
      \begin{tikzpicture}[
          B/.style={pattern=north east lines, pattern color=blue,opacity=0.5},
          G/.style={pattern=north west lines, pattern color=ForestGreen,opacity=0.5},
        ]
        \Diagram(0,0)[ribbons={(G)16ccccrrr, (B)16crrcccr}]{6,5^2,2}
        \draw[->](1.5, 1)node[right]{  $\alpha=(1,2)$} to [out=180,in=90]  (B-1-2.base);
        \draw[->](1.5,-2)node[right]{$f_\alpha=(4,2)$} to [out=180,in=270] (B-4-2.base);
        \draw[red,<->]([xshift=-1mm]B-1-1.south west)
           --node[rotate=90,above]{$l_\alpha=3$}([xshift=-1mm]B-4-1.south west);
      \end{tikzpicture}
    \end{Example}

    This example shows that partitions can either be typed out in full,
    or using exponential notation, which is usually shorter and easier
    to read.

    Most of the commands in this example are \TikZ commands, but most of
    the picture is drawn by the \keyword{\Diagram} command. The top of
    the example defines two \TikZ-styles, \textsf{B} and \textsf{G},
    which are used in the ribbons. The Young diagram is placed at
    position $(0,0)$ by the \keyword{\Diagram} command. The three
    \keyword{\draw} commands in this example are used to add the three
    arrows to the picture. You should consult the very readable \TikZ
    manual for an explanation of how these commands work.

    From the viewpoint of \aTableau, the most important point of this
    example is that the \keyword{\Diagram} command can be used inside a
    \keyword{tikzpicture} environment because it is equipped with the
    Cartesian coordinate $(0,0)$. The example also uses the four named
    coordinates, (\textsf{B-1-1}), (\textsf{B-1-2}), (\textsf{B-4-1}),
    and (\textsf{B-4-2}). When \aTableau draws a tableau, or a diagram, it
    creates \TikZ named coordinates for each of the boxes in the
    diagram, so that (\textsf{B-r-c}) refers to the box in row \textsf{r} and
    column \textsf{c}. Giving finer control, the following standard
    \TikZ \textit{anchors} are available:

    \begin{equation}\label{E:NodeNames}
      \aTabset{align=center}
      \begin{tikzpicture}[every node/.style={font=\tiny, text=gray}]
          \Diagram(0,0)[scale=4]{1}
          \node[label={above:(B-r-c)}] at (B-1-1){$\color{red}\bullet$};
          \node[label={above:(B-r-c.north)}] at (B-1-1.north){$\color{red}\bullet$};
          \node[label={below:(B-r-c.south)}] at (B-1-1.south){$\color{red}\bullet$};
          \node[label={right:(B-r-c.east)}] at (B-1-1.east){$\color{red}\bullet$};
          \node[label={left:(B-r-c.west)}] at (B-1-1.west){$\color{red}\bullet$};
          \node[label={above right:(B-r-c.north east)}] at (B-1-1.north east){$\color{red}\bullet$};
          \node[label={above left:(B-r-c.north west)}] at (B-1-1.north west){$\color{red}\bullet$};
          \node[label={below right:(B-r-c.south east)}] at (B-1-1.south east){$\color{red}\bullet$};
          \node[label={below left:(B-r-c.south west)}] at (B-1-1.south west){$\color{red}\bullet$};
      \end{tikzpicture}
      \qquad
      \begin{tikzpicture}[every node/.style={font=\tiny, text=gray}]
          \Diagram(0,0)[ukrainian, scale=4]{1}
          \node[label={above:(B-r-c)}] at (B-1-1){$\color{red}\bullet$};
          \node[label={above:(B-r-c.north)}] at (B-1-1.north){$\color{red}\bullet$};
          \node[label={below:(B-r-c.south)}] at (B-1-1.south){$\color{red}\bullet$};
          \node[label={right:(B-r-c.east)}] at (B-1-1.east){$\color{red}\bullet$};
          \node[label={left:(B-r-c.west)}] at (B-1-1.west){$\color{red}\bullet$};
          \node[label={above right:(B-r-c.north east)}] at (B-1-1.north east){$\color{red}\bullet$};
          \node[label={above left:(B-r-c.north west)}] at (B-1-1.north west){$\color{red}\bullet$};
          \node[label={below right:(B-r-c.south east)}] at (B-1-1.south east){$\color{red}\bullet$};
          \node[label={below left:(B-r-c.south west)}] at (B-1-1.south west){$\color{red}\bullet$};
      \end{tikzpicture}
    \end{equation}

    In the name name  (\textsf{B-r-c}), $r$ and $c$ should be replaced
    by the row and column index of a box in the tableau, because these
    anchors are \textit{only} defined for boxes in the tableau. These
    \textit{anchors} are a standard part of \TikZ.  If you change the
    shape of the node then the available anchors may change; for more
    information see the \TikZ manual. You can change the \textsf{B} in
    the name of these anchors to anything you like using the
    \option{name} option.

    The previous example also shows that the \keyword{\Diagram} command
    accepts a \option{ribbons} option, which adds one or more
    \textit{ribbons} to the diagram. Both the ribbon and the individual
    boxes in the ribbon can be given \TikZ styling; see
    \autoref{SS:RibbonTableau}. All of the options to these commands are
    described in the following sections.

    In the example above the Cartesian coordinate $(0,0)$ is necessary
    because the \keyword{\Diagram} command is embedded inside a
    \keyword{tikzpicture} environment, but the coordinate itself is not
    important in the sense that changing the coordinate will not change
    the picture. Cartesian coordinates become more meaningful when there
    are more components inside the \keyword{tikzpicture} environment.
    Consider:

    \begin{example}[lefthand width=0.14\textwidth, sidebyside gap=0mm,]
      % \usetikzlibrary{decorations.pathmorphing}
      \begin{tikzpicture}[wave/.style={thick,red,->,decorate,decoration=snake}]
        \aTabset{ribbon style={fill=blue!20, opacity=0.4}}
        \Diagram(0,0)[e=2,show=residues,ribbons={13crc},name=A]{3,2}
        \Diagram(0,-2.5)[e=2,show=residues,ribbons={12rcr}]{2^2,1}
        \draw[wave]([shift={(0,-0.05)}]A-2-1.south east)
                    --([shift={(0,0.05)}]B-1-1.north east);
      \end{tikzpicture}
    \end{example}\label{Ex:ABnames}

    In particular, in the first diagram, notice the use of
    \option{name}\textsf{=A} to make the named coordinates take the form
    (\textsf{A-r-c}), instead of the default of (\textsf{B-r-c}).

    The following sections describe the commands defined by the package
    and how to use them.  We start with the \keyword{\Tableau} command
    because all of the diagram and tableau commands, except for
    \keyword{\RibbonTableau}, are derived from \keyword{\Tableau}.

    Finally, the condition that coordinates are required only when these
    diagrams are being used inside a \keyword{tikzpicture} environment
    needs to be interpreted appropriately. For example, coordinates are
    not required when the diagrams are inside nodes in these diagrams:

    \begin{Example}
    % \usetikzlibrary{matrix}
    \begin{tikzpicture}[arr/.style={->,SteelBlue, thick}]
      \matrix (M)[matrix of nodes,row sep=4mm,column sep=4mm]{
          &    &    & \ShiftedTableau{1234,567} \\
          &    & \ShiftedTableau{1235,467}
          &    & \ShiftedTableau{1243,567}\\
          & \ShiftedTableau{1236,457}
          &    & \ShiftedTableau{1245,367}
          &    & \ShiftedTableau{1253,467}\\
        \ShiftedTableau{1237,456}
          &    & \ShiftedTableau{1246,357}
          &    & \ShiftedTableau{1254,367}\\
          & \ShiftedTableau{1247,356}\\
       };
       \draw[arr](M-1-4)--node[above]{$\sigma_4$}(M-2-3);
       \draw[arr](M-1-4)--node[above]{$\sigma_3$}(M-2-5);
       \draw[arr](M-2-3)--node[above]{$\sigma_5$}(M-3-2);
       \draw[arr](M-2-3)--node[above]{$\sigma_3$}(M-3-4);

    \subsection{Tableaux}
    \indexcmd{Tableau}
    \label{S:Tableau}
    This section describes the \keyword{\Tableau} command, which draws
    tableaux or labelled diagrams. The syntax for this command is:

    \keyword{\Tableau (x,y) [options] {tableau entries}}

    \begin{syntax}
      \item[$(x,y)$]\index{xy@$(x,y)$} The Cartesian coordinates $(x,y)$
      are needed if and only if the tableau is inside a
      \keyword{tikzpicture} environment.
      \item[options] Optional arguments, described below.
      \item[tableau entries]\index{tableau entries} The
      \keyword{tableau entries} are given as a comma separated list, with commas
      separating rows and each \textit{token}, or braced-group of
      tokens, being in a separate column. As we explain below, each
      tableau entry can be prefixed by (optional) \TikZ styling
      commands.
    \end{syntax}

   We show by example how to use the \keyword{\Tableau} command, starting
   with basic usage and finishing with style. Inside \keyword{\Tableau}, the rows
   are separated by commas, with the column entries given by the
   ``letters'' in the entry for each row:

  \begin{example}
    \Tableau{ 1234, 56, 7, 8 }       \qquad
    \Tableau{ \alpha\gamma\delta q,
              xy\eta,
              \lambda\mu,
              \pi\sum }
  \end{example}

  You can put all of the tableau specifications on one line, without any
  spaces. Alternatively, as I do, you can add spaces and line breaks
  between the tableau entries to improve readability (although, you
  cannot have blank lines inside a \keyword{\Tableau} command). As this
  example suggests, by default, the entries of a tableau are typeset as
  mathematics, but this can be changed.

  As the second example above shows, the tableau entries are
  \textit{not} just individual characters: they also include \TeX\
  commands. Tableaux frequently contain entries that are not single
  characters, or given by commands. Such entries can be added to a
  tableau by enclosing them in braces:

  \begin{example}
    \Tableau{ 13{10}{11}{2x}, 2{x^2} }
  \end{example}

  There is one additional caveat for braced-entries: any
  \textit{singleton} braced entry \keyword{{...}} in a row needs to be
  double-braced. Double-bracing if only necessary when you have a row
  with a single column and the entry in that column needs braces.  The
  need for double-bracing of singleton entries is a consequence of how
  \LaTeX3 detects braced commas in comma separated lists.

  The following example shows the difference between single and
  double-bracing for tableau entries.
  \index{tableau entries!double-braced}

  \begin{example}
    \Tableau{ 1345, 8{10}, {11}, {12} } \qquad
    \Tableau{ 1345, 8{10}, {{11}}, {{12}} }
  \end{example}

  The $10$'s in these tableaux do not need to be doubled-braced because
  they are not the only entry in the second row. In contrast, the~$11$
  and~$12$ in the first tableau are both treated as being two characters
  in different columns in the tableau because they are not
  doubled-braced. In the second tableau, the~$11$ and~$12$ occupy only
  one column because they are double-braced.

  \subsubsection{Adding style}
  \index{star}
  \index{*|see{star}}
  \index{tableau entries!style}
  \index{tableau entries!star}

  The \aTableau package provides several different ways to add \TikZ
  style specifications to the entries in tableaux, and to boxes in
  diagrams.  This is made possible by adding style prefixes to the
  tableau entries.  To take advantage of these styles you will need at
  least rudimentary knowledge of how \TikZ-styles work. The examples in
  this manual give a good indication of what is possible. For anything
  more exotic, please consult the \TikZ manual.

  The simplest way to change the style of an entry in a tableau is to
  add a $*$-prefix to the entry in the tableau specifications.

  \begin{example}
    \Tableau{ 1*2*3*4*5, 6*789, {10}*{11}{12}, {{13}} }
  \end{example}

  By default, the $*$-entries are given the \TikZ styling
  \keyword{fill=blue!20}. You can change the default $*$-style using the
  \option{star style} key:


  \begin{example}
    \Tableau[star style={fill=red!20,draw=red,thick}]
            { 1*2*3*4*5, 6*789, {10}*{11}{12}, {{13}} }
  \end{example}

  Notice that the $*$-styling takes precedence over the default styling
  of the surrounding boxes. This is because the boxes with the default
  styling are placed first, in the order that they are entered, after
  which the boxes with custom stylings are placed.

  The $*$-syntax is a quick shorthand for adding emphasise some of the
  boxes in a tableau using a \textit{single} style. It is possible to
  give every box in the tableau its own style by putting \TikZ-style
  specifications inside square brackets, \keyword{[...]}, before the
  corresponding letter in the tableau specification.  You can mix the
  $*$-syntax and the \keyword{[...]}-style syntax for different entries,
  although you can only apply one of these style settings to any given
  box.

  \begin{example}
    \Tableau[french]{
        1 [blue!20]2 [circle]3 [red]4 [cyan]5,
        6 *7 8 9,
        {10} *{11} {12},
        {{13}} }
  \end{example}

  It is not necessary to add spaces between the entries like this. This
  example puts spaces between each entry to make it clearer how the
  style specifications are applied to the boxes in the tableau.

  Omitting some technicalities, each box in a tableau is constructed as
  a \TikZ node, so any \TikZ-style settings for a node can be used.
  This example shows that some care is needed when changing the style
  because simply giving a colour will change both the colour used to
  fill the box and the text colour, which is why some of tableau entries
  in the last example appear to be blank.  If you are already familiar
  with \TikZ then you probably already know what to do. If not, then
  here is a short list of useful style specifications for \TikZ nodes:
  \begin{center}\index{fill}\index{text}\index{draw}
     \begin{tabular}{ll}\toprule
       Style & Meaning \\\midrule
       \keyword{text}$=\langle\text{colour}\rangle$
            & Sets the text colour in the box \\
       \keyword{fill}$=\langle\text{colour}\rangle$
            & Sets the fill, or background, colour of the box\\
       \keyword{draw}$=\langle\text{colour}\rangle$
            & Sets the boundary colour of the box\\
       \bottomrule
     \end{tabular}
  \end{center}
  In the style settings, any valid \LaTeX\ colour name can be used; see, for
  example, the \ctan{xcolor} manual. In addition, you can change the
  border thickness, and its properties, with the style specifications
  \keyword{thin},
  \keyword{thick},
  \keyword{very thick},
  \keyword{ultra thick},
  \keyword{dashed},
  \keyword{dotted}, $\dots$.

  If you want to apply ``complicated'' style settings, such as
  \keyword{draw=cyan,ultra thick} which consists of a comma separated
  list of $\TikZ-$style settings, then the entire style must be put
  inside matching square and curly braces \keyword{[{...}]}.  This is
  necessary to ensure that  \LaTeX\ does not get confused by the commas
  in the style setting because commas are used to separate the rows of
  the tableau.

  \begin{example}
    \Tableau[australian]{
        1 [fill=blue!20]2 3 [text=red]4 5,
        [{circle,fill=orange}]6 *7 8 9,
        [{draw=cyan,ultra thick}]{10} *{11} {12},
        [{dashed,draw=red,ultra thick}]{13} }
  \end{example}

  The last example shows that it is not necessary to double-brace
  singleton entries when they have a style specification.

  This example shows that modifying the borders of a box does not add
  much emphasis, so use this sparingly, especially because any boxes
  placed later are likely to redraw the boundary (see
  \autoref{SS:order}). In addition, the orange circle in this example
  looks too big, but it is what we asked for because its diameter matches
  the horizontal width of the box. The next example shows that we get a
  better result for \textit{Australian} and \textit{Ukrainian} tableaux
  if we add a \textit{minimum size} specification.

  For complicated styles, or styles that are used often, we recommend
  defining the style outside of the \keyword{\Tableau} command by using
  the \keyword{\tikzset} command:

  \begin{example}[lefthand width=0.3\textwidth]
    \tikzset{
       B/.style={fill=blue!40, text=yellow},
       G/.style={fill=green!80!blue,circle,minimum size=5mm},
       R/.style={fill=red,text=white, font=\tiny},
       Y/.style={fill=yellow!20, text=blue}
    }
    \Tableau[french]{[R]1[B]2,[G]3,[Y]4} \quad
    \Tableau[ukrainian]{[R]1[B]2,[G]3,[Y]4} \newline
    \Tableau[english]{[R]1[B]2,[G]3,[Y]4} \quad
    \Tableau[australian]{[R]1[B]2,[G]3,[Y]4}
  \end{example}

  In this example, the \TikZ-styles \textsf{B}, \textsf{G} \textsf{R},
  and \textsf{Y} are applied to the tableaux entries labelled $1$, $2$,
  $3$, and $4$, respectively. The colours used here, and throughout this
  manual, are for demonstration purposes only: I recommend against such
  garish choices in any self-respecting document!

  \subsubsection{Tableau coordinates}
  \index{xy@$(x,y)$}
  \index{Cartesian coordinates}

  The $(x,y)$-coordinates are needed if, and only if,
  \keyword{\Tableau}command is used inside a \keyword{tikzpicture}
  environment, in which case $(x,y)$ gives the coordinates of the
  ``outside corner'' of the box in row~$1$ and column~$1$ of the
  tableau. The following example shows how tableaux are placed using
  $(x,y)$-coordinates inside a \keyword{tikzpicture} environment. The example
  also shows that, by default, the tableau boxes are half a unit wide.

  \tikzset{
    addgrid/.code = {
      % draw coordinate axes
      \draw[help lines, step=0.5,thin, LightGrey](-0.5,-1.5) grid (4,2.5);
      \foreach \pt in {0,...,3} {
        \node[below,font=\tiny] at (\pt,-1.5){$\pt$};
      }
      \foreach \pt in {-1,...,2} {
        \node[left, font=\tiny] at (-0.5,\pt){$\pt$};
      },
    }
  }

  \begin{example}
    \begin{tikzpicture}[addgrid]
      \Tableau(0,0)    [english]   {123,45}
      \Tableau(0,0.5)  [french]    {123,45}
      \Tableau(2.5,0.5)[ukrainian] {123,45}
      \Tableau(2.5,0)  [australian]{123,45}
    \end{tikzpicture}
  \end{example}

  \subsubsection{Tableau options}
  \label{SS:TableauOptions}

  The optional $*$-styles and $[\ldots]$-styles are the main mechanism
  for styling individual nodes in a tableau. Using a key-value syntax,
  you can change other aspects of the tableaux produced by the
  \keyword{\Tableau} command.  The rest of this section describes these
  keys, their default values, and gives examples of how to use them.

  The options described below can be used with the \keyword{\Tableau}
  command by placing them as a comma separated list inside square
  brackets:
  \begin{center}
    \keyword{\Tableau[options] {tableau entries}}\quad or \quad
    \keyword{\Tableau(x,y) [options] {tableau entries}}.
  \end{center}
  The options can appear in any order, with later options having
  precedence over earlier ones.

  When used inside a \keyword{\Tableau} command, the options only affect
  that particular tableau. Most of these options, or settings, can also
  be used in the \keyword{\aTabset} command, or as optional arguments in
  \keyword{\usepackage[..]{atableau}}, to change the default settings of
  all tableaux in the same \LaTeX\ group.  For example, if you put the
  command \keyword{\aTabset[ukrainian]} in the preamble of your document
  then, by default, latter tableaux and Young diagrams will be drawn
  using the Ukrainian convention.

  \option[english]{english}\\
  \option*[]{french}\\
  \option*[]{ukrainian}\\
  \option*[]{australian}

  These four options change the convention, or orientation that is used
  to draw the tableaux. Rather than define this precisely, we use the
  tired and true method in combinatorics of defining by example.

  \begin{example}[lefthand width=0.3\textwidth]
    \tikzset{R/.style={fill=red!20,draw=red}}
    \Tableau[ukrainian]{12[R]3,45}
  \end{example}

  \begin{example}[lefthand width=0.3\textwidth]
    \tikzset{R/.style={fill=red!20, circle, draw=red, ultra thick}}
    \Tableau[french]{12[R]3,45}
  \end{example}

  \begin{example}[lefthand width=0.3\textwidth]
    \tikzset{B/.style={fill=blue!20, dashed, thick}}
    \Tableau[english]{12[{B, thick}]3,45}
  \end{example}

  \begin{example}[lefthand width=0.3\textwidth]
    \tikzset{B/.style={fill=blue!20, thick, text=white}}
    \Tableau[australian]{12[{B, thick}]3,45}
  \end{example}

  By default, the \option{english} convention is used to draw tableaux
  and Young diagrams. This can be changed using \keyword{\aTabset}. The
  capitalised names, \keyword{Australian}, \keyword{English},
  \keyword{French} and \keyword{Ukrainian}, are also recognised. If, for
  example, your document is only going to \option{french} tableaux and
  diagrams, then you can specify this when you load the package using
  \keyword{\usepackage[french]{atableau}}.

  The \keyword{\Tableau} settings used as options to the
  \keyword{\usepackage} statement change the default settings throughout
  the document.

  \option[bottom]{align}[centre/bottom/top]

  Use the \option{align} key to change the vertical centering of
  tableaux. By default, tableaux are aligned on their baseline.

  \begin{example}
    \Tableau{123,455,674} \qquad
    \Tableau{123,455}
  \end{example}

  \begin{example}
    \Tableau[align=top]{123,455,674} \qquad
    \Tableau[align=top]{123,455}
  \end{example}

  \begin{example}
    \aTabset{align=bottom}
    \Tableau{123,455,674} \qquad
    \Tableau{123,455}
  \end{example}

  \begin{example}
    \aTabset{align=centre}
    \Tableau{123,455,674} \qquad
    \Tableau{123,455}
  \end{example}

  It is not necessary to have the same \option{align} keys on each
  tableau. Here, and elsewhere, American spelling variations are
  tolerated.

  \option[0.5]{box height}[A decimal giving the height in \unit{cm}]\\
  \option*[0.5]{box width}[A decimal giving the width in \unit{cm}]

  Sets the width and height of the boxes in the tableau. The default
  height and width of the boxes is \qty{0.5}{cm} when using the
  \option{english} and \option{french} conventions and
  \qty{0.7012}{cm}, which is approximately $1\sqrt{2}$\unit{cm}, when
  using the \option{ukrainian} and \option{australian} conventions. (In
  all cases, the side length of the squares is \qty{0.5}{cm}.)

  \option[false]{conjugate}[false/true]

  Draws the \emph{conjugate} tableau, where the rows and columns are
  swapped.

  \begin{example}
    \Tableau{123,45,67}    \qquad
    \Tableau[conjugate]{123,45,67}
  \end{example}

  \option[B]{name}[string/character]

  By default, the boxes can be referenced using the node names
  \textsf{(B-1-1)}, \textsf{(B-1-2)}, \textsf{(B-1-3)}, \dots, with
  \textsf{(B-$r$-$c$)} referring to the node in row~$r$ and column~$c$.
  This feature is most useful when you have several tableaux inside a
  \keyword{tikzpicture} environment because if you only have one
  tableau, then it is unlikely that you need to change the default
  prefix for the node names. Examples are given in \autoref{Ex:ABnames}
  and \autoref{Ex:pointing}.

  See the introduction to this section for examples and for more
  information about using node names and anchors.

  \option[]{tikz before}[\TikZ commands]\\
  \option*[]{tikz after}[\TikZ commands]

  These commands inject \TikZ code  into the underlying
  \keyword{tikzpicture} environment,  where the tableau is constructed, before and after the
  \keyword{\Tableau} command, respectively.

  \begin{example}[lefthand width=0.24\textwidth]
   \Tableau[
     tikz before={\draw[help lines,step=0.5](-0.5,-2)grid(2,0.5);},
     tikz after={\draw[ultra thick,red]
                   (B-1-3.north east)--(B-1-3.south east)
                   --(B-2-1.north east)--(B-3-1.south east)
                   --(B-3-1.south west);
     }]{134,567,89}
  \end{example}

  Both before and after hooks are provided because the order in which
  you place \TikZ commands can affect the drawing, primarily because
  later commands are drawn over the top of earlier ones. Note that the
  \textit{before} code cannot use the named nodes \textsf{B-r-c} because
  this code is executed before the tableau is drawn. For examples, see
  \autoref{SS:order}.

  \option[]{tikzpicture}[\TikZ-style commands]

  Use this command to add an optional argument to the underlying
  \keyword{tikzpicture} environment that contains the tableau. The
  \option{tikzpicture} option is ignored  when the \keyword{\Tableau}
  command is equipped with $(x,y)$-coordinates.

  \begin{example}[lefthand width=0.3\textwidth]
    \Tableau[ukrainian, name=A,
      tikzpicture={remember picture}]
      { 1[fill=blue!20]23[text=red]45,
        6*78[{draw=cyan,ultra thick}]9,
        [{circle,fill=orange}]{10}*{11}{12},
        [{dashed,draw=red,ultra thick}]{13}
      }
  \end{example}\label{Ex:pointing}

  Since \option{name}=A, the nodes in this tableau are referenced as
  \textsf{(A-1-1)}, \textsf{(A-1-2)}, \textsf{(A-1-3)}, \dots. As the
  example also uses \option{tikzpicture}=\textit{remember picture}, we
  can reference the node names in last tableau from other
  \keyword{tikzpicture} environments. For example, we can do things like
  this:

  \vspace*{3mm}

  \begin{example}[lefthand width=0.3\textwidth]
     \tikz[remember picture, overlay]
       \draw[very thick, ->,red]
         (0,0) node[right,align=left]
            {This circle is too big!\\Use minimum size=5mm}
         to [out=135, in=225](A-3-1);
  \end{example}

  As a second example, we can use \keyword{tikzpicture=rotate=30} to try
  to rotate a tableau by 90 degrees with:

  \begin{example}
   \Tableau[tikzpicture={rotate=90}]{123,45} \qquad
   \begin{tikzpicture}[rotate=90]
     \Tableau(0,0){123,45}
   \end{tikzpicture}
  \end{example}

  The two tableaux constructions in this example are equivalent. An
  oddity with both of these examples is that the numbers have not been
  rotated, which is because \emph{rotate} does not affect component
  objects, such as nodes, inside a \TikZ picture. You can rotate
  everything in the picture by using \textit{transform canvas}:

  \vspace*{15mm}

  \begin{example}[valign=top, lefthand width=0.3\textwidth]
    \Tableau[tikzpicture={transform canvas={rotate=90}}]{123,45}
  \end{example}

  \option[]{ribbons}[a comma separated list of ribbon specifiers]
  \option*[]{snobs}  [a comma separated list of ribbon specifiers]
  \option*[fill={blue!20}]{ribbon style}[\TikZ-style commands]

  Use the \option{ribbons} key to add a comma separated list of ribbons
  to your tableau. The syntax for adding ribbons is described in
  \autoref{SS:RibbonTableaux}, which describes the
  \keyword{\RibbonTableau} command. The basic idea is that you specify
  the location of the head of the ribbon, which has minimum row index
  and maximum column index, and then gives a sequence of \textsf{r}'s
  and \textsf{c}'s depending on whether the ribbon moves along a row or
  column -- and, of course, you can add style. Use the
  \option{ribbon style} option to change the default style of the
  ribbons. For example, noting that a box is a ribbon of length~$1$, the
  following code uses ribbons of length one (that is, nodes), to
  highlight the addable nodes of this tableau.

  \begin{example}
    \Tableau[star style={fill=green!80!blue,text=white},
            ribbon style={fill=yellow},
            ribbons={15,24,42,51},
    ]{123*4,567,89*{10},*{11}}
  \end{example}

  The ribbons are always assumed to be contained inside the diagram of
  the tableau, so they will not change the  boundary pf the tableau.
  Use \option{snobs} to add ribbons to the tableau that are placed
  \textit{after} the border of the diagram has been drawn. (Writing
  \textit{ribbons} backwards gives \textit{snob}bir.) The default style
  for a \keyword{snobs} ribbon is given by \keyword{ribbon style}.

  \begin{example}
    \Tableau[star style={fill=green!80!blue,text=white},
             ribbon style={fill=yellow},
             snobs={15,24,42,51},
    ]{123*4,567,89*{10},*{11}}
  \end{example}

  The difference between these two pictures is that in the second
  picture the nodes in the \option{snobs} are drawn over the tableau
  boundary, causing the boundary to disappear for these nodes.  For this
  reason, in most cases you should use \option{ribbons}, and avoid
  \option{snobs}. I do not know of a real example where I would use
  \option{snobs}, so I would be pleased to hear of any example where it
  is necessary.

  \option*[0.5]{box width}{a decimal number (in centimetres)}

  \option*[1]{xscale}[a decimal number]
  \option*[1]{yscale}[a decimal number]

  By design, boxes in tableaux do not automatically resize to fit
  their contents. For example, consider:

  \begin{example}
    \Tableau{ 13{10}{11}{1+x}, 2{1+x} }
  \end{example}

  This is unlikely to be the desired output! The options \option{scale},
  \option{xscale} and \option{yscale} can be used to rescale the boxes
  in tableaux and diagrams. The names are slightly misleading because
  \option{xscale} rescales in the direction of increasing column index
  and \option{yscale} rescales in the direction of increasing row index.

  \begin{example}
    \Tableau[scale=2]{ 13{10}{11}{1+x}, 2{1+x} }
  \end{example}
  \begin{example}
    \Tableau[xscale=2]{ 13{10}{11}{1+x}, 2{1+x} }
  \end{example}
  \begin{example}
    \Tableau[ukrainian, xscale=2]{ 13{10}{11}{1+x}, 2{1+x} }
  \end{example}

  \begin{danger}
    The options, \option{scale}, \option{xscale}, and \option{yscale},
    affect all \aTableau diagrams, including tableaux, Young diagrams
    and abacuses. If you want to use \keyword{\aTabset} to change the
    sizes of all diagrams and tableaux in your document it may be better
    to use the \option{box height} and \option{box width} options.
  \end{danger}

  \begin{example}
    \Tableau[star style={blue}]{..*., .*.*., .*.}
  \end{example}

  \begin{example}
    \tikzset{R/.style={fill=blue!20,draw=red,thick}}
    \Tableau{12[R]3,4[R]5}
    \Tableau[star style={fill=blue!50,text=white}]{1*23,4*5}
  \end{example}

  \subsubsection{Compositions}
  \index{composition}

  As we have defined them, tableaux are always of \textit{partition}
  shape, in that the lengths of the rows form a weakly decreasing
  sequence. In fact, the \keyword{\Tableau} command also accepts
  tableaux of \emph{composition} shape, where the lengths of the rows
  are not necessarily in decreasing order

  \begin{example}
    \Tableau{ 245,82751,31 }
    \SkewTableau{1,2}{ 245,827,231 }
  \end{example}

  Empty rows in tableaux and Young diagrams are not supported!

  \subsubsection{Drawing order}
  \label{SS:order}
  \index{tableau!drawing order}

  When \TikZ constructs a picture, later objects are drawn over the top
  of earlier ones, which means that earlier features can be obscured or
  lost. For this reason it is useful to know the drawing order used by
  \keywords{\Tableau}:
  \begin{itemize}
    \item First, the tableau entries that use the default styling are
    placed, in the order that appear in the \textit{tableau
    specifications}.
    \item Secondly, the styled tableau entries are placed, in the order
    that they appear in the \textit{tableau specifications}.
    \item Next, the nodes in any \option{ribbons} are placed, again in
    the order that the ribbons are listed. For each ribbon, first the
    border of the ribbon is drawn, together with any styling for the
    ribbon, and then the boxes in the ribbon, together with any styling,
    are added.
    \item The boundary of the tableau is drawn next.
    \item Finally, the \option{snobs} are are drawn, in the order that they are
    specified.
  \end{itemize}

  \subsubsection{Special characters}
  \index{tableau!special characters}
  \index{tableau!comma}
  \index{tableau!*}
  \index{tableau![}

  The characters  \keyword{[]}, \keyword{,}, and \keyword{*} have special
  meanings in the \textit{tableau specifications}. If you want these
  characters to be entries in a tableau, then you need to enclose them
  in braces:

  \begin{example}
    \Tableau{ 13{,}5], 8{[}, {{*}} }
  \end{example}

  There is no need to enclose \keyword{]} in braces because it only
  becomes special when it is preceded by a \keyword{[} character.

  \subsection{Young diagrams}
  \label{S:diagram}
  \indexcmd{Diagram}
  \index{young diagram@Young diagram|see{diagram}}
  \index{diagram}
  \index{diagram}
  \index{partition!diagram}

  A \emph{Young diagram}, or simply a \emph{diagram}, is an unlabelled
  tableau. Diagrams can be drawn using the \keyword{\Tableau} command:

  \begin{example}
    \Tableau{~~~,~~~,~~,~}
  \end{example}

  This approach works but it is slightly cumbersome and hard to
  proofread, so \aTableau provides the \keyword{\Diagram} command.  The
  \keyword{\Diagram} command uses almost the same syntax as the
  \keyword{\Tableau} command:

  \keyword{\Diagram (x,y) [options] {partition}}

  As with the \keyword{\Tableau} command, the $(x,y)$-coordinates are
  needed if and only if the diagram is inside a \keyword{tikzpicture}
  environment.  In the \keyword{\Diagram} command, partitions can be
  specified either as a comma separated list of weakly decreasing
  nonnegative integers, or using \emph{exponential notation} for partitions:
  \index{exponential notation}
  \index{diagram!exponential notation}

  \begin{example}
    \aTabset{align=centre}
    \Diagram{4,3^2,1^3} \qquad
    \Diagram{4,3,3,1}
  \end{example}

  Internally, \keyword{\Diagram} actually does something like the first
  example in this section, so all of the options for the
  \keyword{\Tableau} command can be used with \keyword{\Diagram}. We
  do not repeat all of these options in this section and, instead,
  highlight some of the more interesting ones together with new
  options that apply specifically to diagrams.

  \begin{samepage}
  We start by advertising that \keyword{\Diagram} supports the four
  different conventions for diagrams, with \option{english} being the
  default:

  \option[english]{english}
  \option*[]{french}
  \option*[]{ukrainian}
  \option*[]{australian}
  \end{samepage}

  \begin{example}
    \Diagram[australian]{3,2,1} \qquad
    \Diagram[australian]{3,1^3}
  \end{example}

  \begin{example} % English is the default convention
    \Diagram[english]{3,2,1} \qquad
    \Diagram{3,1^3}
  \end{example}

  \begin{example}
    \Diagram[french]{3,2,1} \qquad
    \Diagram{3,1^3}
  \end{example}

  \begin{example}
    \Diagram[ukrainian]{3,2,1} \qquad
    \Diagram{3,1^3}
  \end{example}

  \option[true]{border}[true/false]\\
  \option[false]{no border}[true/false]\\
  If \option{border} is true, then the border of the tableau is drawn.
  If \option{border} is false, then the border of the tableau is not
  drawn. By default, the border is drawn. The \option{no border} option,
  which is is provided only for completeness, is the inverse of
  \option{border}. That is, \option{no border} is equivalent to
  \option{border=false}, and \option{no border=false} is equivalent to
  \option{border=true}.

  \begin{example}
    \Diagram[border=false]{3,2,1} \qquad
    \Diagram[no border]{3^2,1}
  \end{example}

  \option[false]{border only}[false/true]
  If \option{border only} is true, then only the border of the tableau
  is drawn, and not the internal walls around the boxes.

  \begin{example}
    \Diagram[border only]{3,2,1}
  \end{example}

  \option[false]{conjugate}[false/true]
  Draws the conjugate diagram, which has the rows and columns
  interchanged.

  \begin{example}
    \Diagram[conjugate]{3,2,2} \qquad
    \Diagram[ukrainian, conjugate]{2,1^3}
  \end{example}

  \option[]{ribbons}[ribbon specification]\\
  \option[]{snobs}[ribbon specification]

  \option[]{show}[contents,final,hooks,initial,residue]
  \index{contents}
  \index{diagram!show}
  \index{initial tableau}
  \index{tableau!initial}
  \index{tableau!final}
  Use \option{show} option to draw particular types of tableau of the
  specified shape. The possible choices are:
  \begin{itemize}
    \item\option{show=contents} print the diagram where each box is
    labelled by its \emph{content}, which is the row index minus the
    column index.

    \begin{example}
      \Diagram[show=contents]{3,2,1} \qquad
      \Diagram[french,show=contents]{2^2,1}
    \end{example}

    \item\option{show=final} print the \emph{final tableau} of this
    shape, which has the numbers $1,2\dots,n$ entered in order from top
    to bottom down successive columns (with appropriate modifications for
    the non-English conventions).

    \index{final tableau}
    \index{tableau!final}
    \begin{example}
      \Diagram[show=final]{3,2,2} \qquad
      \Diagram[australian,show=contents]{2^2,1}
    \end{example}

    \item\option{show=hooks} print the diagram where each box is
    labelled by the corresponding \emph{hook length}. If $\lambda$ is a
    partition and $\lambda'$ is its conjugate partition then the hook
    length of the box in row~$i$ and column~$j$ is
    $\lambda_i-i+\lambda'_j-j+1$.

    \index{tableau!hook lengths}
    \index{hook lengths}
    \begin{example}
      \Diagram[show=hooks]{3,2,2} \qquad
      \Diagram[ukrainian,show=hooks]{2^2,1}
    \end{example}

    \item\option{show=initial} print the \emph{initial tableau} of this
    shape, which has the numbers $1,2\dots,n$ entered in order from top
    left to right along down successive rows (with appropriate modifications for
    the non-English conventions).

    \index{initial tableau}
    \index{tableau!initial}
    \begin{example}
      \Diagram[show=initial]{3,2,2} \qquad
      \Diagram[ukrainian, show=initial]{3^2,2}
    \end{example}

    \item\option{show=residues} print the diagram where each box is
    labelled by its \emph{residue}, which is the row index minus the
    column index modulo some integer~$e$, which must also be supplied.

    \index{residue}
    \index{tableau!residues}
    \begin{example}
      \Diagram[show=residues, e=2]{3,2,1} \qquad
      \Diagram[french,show=residues, e=3]{3,2,1}
    \end{example}

  \end{itemize}

  \subsection{Skew tableau and skew diagrams}
  \indexcmd{SkewTableau}
  \indexcmd{SkewDiagram}
  \index{diagram!skew}
  \index{partition!skew}
  \index{tableau!skew}

  A partition $\mu$ is \emph{contained} in a partition $\lambda$, which
  is written as~$\mu\subseteq\lambda$, if $\mu_k\le\lambda_k$, for
  $k\ge0$. If $\mu\subseteq\lambda$ then the \emph{skew partition}
  $\lambda/\mu=\set{(r,c)\in\lambda|(r,c)\notin\mu}$ is the set of nodes
  that are in~$\lambda$ and not in~$\mu$. In this manual, we call $\mu$
  the \emph{inner shape} and $\lambda/\mu$ the outer shape. A \emph{skew
  tableau} is a labelling of the nodes in the diagram of a skew
  partition. Skew partitions and skew tableau can be drawn using the
  commands:

  \keyword{\SkewDiagram (x,y) [options] {inner shape} {outer shape}}\\
  \keyword{\SkewTableau (x,y) [options] {inner shape} {skew tableau}}

  As with the previous commands, the $(x,y)$--coordinates should be given
  if and only if the picture is inside a \keyword{tikzpicture}
  environment.  The inner and outer shapes can either be given as a
  comma separated list of integers or in exponential notation. In a
  \keyword{\SkewTableau}, the tableau entries are specified exactly as
  in the \keyword{\Tableau} command. In particular, they can be given
  optional style commands.
  \index{exponential notation}

  All of the options for the \keyword{\Diagram} command can be used with
  the \keyword{\SkewDiagram} command and, similarly, all fo the options
  for the \keyword{\Tableau} command can be sued with the
  \keyword{\SkewTableau} command.

  \begin{example}
    \tikzset{R/.style={fill=red!20,circle}}
    \SkewTableau{2,1^2}{12[R]3,45,67}
    \SkewTableau[french,skewwall]{2,1^2}{12[R]3,45,67}
  \end{example}

  \subsection{Tabloids}
  \indexcmd{Tabloid}

  A \emph{tabloid} is an equivalence class of tableau where two tableaux
  are equivalent if they have the same entries in each row, up to a
  permutation. In the literature, tabloids are usually drawn with lines
  above and below each row, and no side boundary walls. Tabloids can be
  drawn with the \keyword{\Tabloid} command.

  \keyword{\Tabloid (x,y) [options] {partition }}\\

  The \keyword{\Tabloid} command functions in the same way as the
  \keyword{\Tableau} command, except that it draws tabloids.

  \begin{example}
    \tikzset{R/.style={fill=red!20,circle,minimum size=5mm}}
    \Tabloid[ukrainian]{12[R]3,45}
    \Tabloid[english]  {12[R]3,45}
    \Tabloid[french]   {12[R]3,45}
  \end{example}

  Some authors work with \emph{column tabloids}. Such tableaux do not
  have a dedicated \aTableau command, however, they can be drawn using the
  \option{conjugate} option.  \index{tabloids!column}

  \begin{example}
    \Tabloid[conjugate]{123,4} \qquad
    \Tabloid[conjugate]{~~~,~}
    \Diagram[conjugate,tabloid]{3,1}
  \end{example}

  Similarly, skew tabloids can be drawn using the \option{skew}
  option.
  \label{tabloids!skew}

  \begin{example}
    \aTabset{align=center}
    \Tabloid[skew={2,1}]{1379{11},249{10},6,} \qquad
    \Tabloid[conjugate,skew={1^2}]{1379{11},249{10},6,}
  \end{example}


  \subsection{Ribbon tableaux}\label{SS:RibbonTableaux}
  \indexcmd{RibbonTableau}
  \index{ribbon!head}
  \index{tableau!ribbon tableau}

  A \emph{ribbon} in a tableau, or in a diagram, is a connected strip of
  boxes~$R$ that are totally ordered by their contents. Recall from
  \autoref{S:diagram} that the content of $(a,b)$ is $b-a$. Therefore, a
  ribbon does not contain a
  \Diagram[scale=0.3,border=false,align=bottom]{2^2}-square, and if
  $(a,b)\in R$ then at most one of $(a+1,b)$ and $(a,b-1)$ is in~$R$.
  The \emph{head} of a ribbon~$R$ is the unique node $(a,b)\in R$ such
  that $a'>a$ and $b'<b$ whenever $(a',b')\in R$ and $(a',b')\ne(a,b)$.

  \keyword{\RibbonTableau (x,y) [options] {ribbon specifications}}\\

  Like the other commands, the $(x,y)$-coordinates are optional and are
  required if and only if the diagram is being used as part of a
  \keyword{tikzpicture} environment. All of the options for the
  \keyword{\Tableau} command can be used for the ribbon command, so we
  refer to \autoref{S:Tableau} for a description of the available
  options.

  \subsubsection{Ribbon specifications}
  Unlike the \keyword{\Tableau} command, the entries of a ribbon tableau
  are given by ribbon specifications (rather than tableau)
  specifications. Ribbon specifications are also used in the
  \option{ribbons} and \option{snobs} options discussed in
  \autoref{SS:TableauOptions}.

  To understand the ribbon specifications, observe that
  if $(a,b)$ is the head of a ribbon, then we can walk along the ribbon
  by specifying whether the \textsf{r}ow index increases or the
  \textsf{c}olumn index decreases. That is, the ribbon is uniquely
  determined by specifying its head $(a,b)$ together with a sequence of
  $\textsf{r}$'s and \textsf{c}'s to say whether the row index increases
  or the column index decreases. For example, vertical and horizontal
  ribbsons are given by a sequences of \textsf{r}'s and \textsf{c}'s,
  respectively:

  \begin{example}
    \aTabset{align=top, no border}
    \RibbonTableau{11rrrr} \qquad
    \RibbonTableau{16ccccc}
  \end{example}

  The boundary of a ribbon tableau is the smallest (skew) partition that
  contains all of the ribbons.  As the last example shows, the
  \option{no border} disables the tableau borders because, otherwise, we
  would not be able to see the ribbon boundary in this example because
  the ribbon is the full diagram.

  The following diagram shows a see-sawing ribbon with head in row~$1$
  and column~10, and tail in row~$5$ and column~$1$.

  \begin{example}
    \RibbonTableau{1{10}crcrcrcrccccc}
  \end{example}

  As with tableau entries, each box in a ribbon can be preceded with a
  style specification, either using $*$ for a \option{star style} or
  using $[...]$ for more customised style specifications.

  \begin{example}
    \RibbonTableau{
        *1{10}c*rc*rc*rc*rc*cc*cc,
        *18c[fill=red]rc[fill=blue]rc*rc*cc[fill=teal]c,
        [fill=green]16c*rc*rc*cc,
        *14c[{draw=purple,ultra thick}]cc[fill=orange]r,
        [{fill=yellow,draw=red,ultra thick}]23
    }
  \end{example}

  In particular, the yellow box in this example highlights that boxes
  are ribbons of length~$1$. Most of the time, you want to style the
  entire ribbon rather than styling the individual boxes in the ribbon.
  Optionally, style for the entire ribbon can be given inside
  parentheses $(...)$ at the \textit{start} of the ribbon. Unlike the
  other style specifications, the $*$-shorthand for the \option{star
  style} cannot be applied to an entire ribbon.

  \begin{example}
    \RibbonTableau{
        (fill=red)1{10}crcrcrcrccccc,
        (fill=orange)18crcrc*rcccc,
        (fill=yellow)16crcr[cyan]ccc,
        (fill=green)14cccr,
        (fill=blue)23
    }
  \end{example}

  Any styles for a ribbon are applied to the region bounded by  the
  closed path given by the boundary of the ribbon. The ribbon style is
  applied first, after which any styled boxes in ribbon are drawn. So,
  as this example shows, the optional style of any box in the ribbon
  takes precedence over the style of the full ribbon.

  Finally, it is often necessary to label some of the boxes in the
  ribbon. Optionally, text can be added to the boxes in a ribbon by
  supplying it as a subscript to the corresponding entry in the ribbon
  specification.

  \begin{example}
    \RibbonTableau[french]    { 12_1 c_2 r_3 } \quad
    \RibbonTableau[ukrainian] { 12_a c_b r_c } \quad
    \RibbonTableau[australian]{ 12_{1_a} c_{2_b} r_{3_c} }
  \end{example}

  This example adds spaces between the entries for the different boxes
  in the ribbon for clarity. There are situations where you might to put
  a label of a box on the boundary between two boxes in the diagram.
  There is no dedicated syntax for doing this, however, it is not
  difficult to do this using the named anchors of~\autoref{E:NodeNames}.

  \begin{example}
    \RibbonTableau[shifted, tikz after={
        \node at (B-1-2.east){$1$};
        \node at (B-2-2.east){$2$};
    }]{ 14ccc, 23c }
  \end{example}

  We have now seen the full ribbon specifications. To summarise:
  \begin{itemize}
    \item The ribbon specification starts with optional \TikZ styling
    for the entire ribbon, which is enclosed in $(...)$.
    \item The first entry in the ribbon is given as
    \textsf{a}\textsf{b}, if the head of the ribbon is in row
    \textsf{a} and column \textsf{b}. The remaining entries in the
    ribbon are specified by a sequence of \textsf{r}'s and \textsf{c}'s,
    depending on whether the next box in the ribbon is given by
    increasing the row index, or decreasing the column index,
    respectively.
    \item Optionally, each box in the ribbon, including the head, can be
    given \TikZ-styling by prefixing it with a $*$, which gives the
    ribbon \option{star style}, or by giving \TikZ-style commands inside
    square brackets $[...]$.
  \end{itemize}

  Finally, when drawing the border, ribbons in a
  \keyword{\RibbonTableau} are assumed to be contained in the smallest
  Young diagram that contains all of the ribbons. This said, it is up to
  you to ensure that the boxes in the ribbon not have negative row or
  column indices. Inside a \keyword{\Diagram} or \keyword{\Tableau}
  command, any ribbons added using \option{ribbons}, or \option{snobs},
  do not affect the border of the diagram or tableau, and  it is up to
  you to ensure that the boxes in the ribbon are inside the diagram or
  tableau.  This said, it is a useful feature to be able to place boxes
  outside of the diagrams using \option{ribbons} and \option{snobs}. For
  example, the following picture uses \option{ribbons} to highlight the
  \emph{addable} and \emph{removable} nodes of the partition~$(4,3^2)$.

  \begin{example}
    \tikzset{A/.style={fill=green!10, text=teal},
      R/.style={fill=yellow!10, text=red}}
    \Diagram[ribbons={[A]15_A,[A]24_A,[A]41_A, [R]14_R,[R]33_R}]{4,3^2}
  \end{example}

  We close this section showing how to draw some tableaux that appear
  in the literature.

  \begin{example}
    \RibbonTableau[skew={2^2,1^2},skewwall]{

  \begin{example}

  \section{The code}
  Every command provided by \aTableau
  uses \TikZ, with the bulk
  of the code in the package being written in \LaTeX3. Most of the tableau
  commands are special cases of the \keyword{\Tableau} command. Here is
  a dictionary:
  \begin{center}
     \begin{tabular}{ll}\toprule
           Command & Equivalent \keyword{\Tableau} command \\\midrule
           \keyword[width=120mm]{\ShiftedTableau{...}}
                & \keyword{\Tableau[shifted]{...}}\\
           \keyword{\SkewTableau{$\mu$}{...}}
                & \keyword{\Tableau[skew=$\mu$]{...}}\\
           \keyword{\Tabloid}\{...\}
                & \keyword{\Tableau[tabloid]{...}}\\
           \bottomrule
