    This package provides commands for drawing Young diagrams, tableaux,
    abacuses and other common objects used in algebraic combinatorics
    and representation theory.  The commands are designed to be easy to
    use and easy to read.  The literature contains many embellishments
    of the basic diagrams, so we provide mechanisms to customise and
    style these diagrams.

    The package provides basic commands for drawing these objects,
    together with styling options and a key-value interface for adding
    extra features. Under the hood, everything is drawn using \TikZ.
    It is possible to use all of our commands either as standalone
    commands, or as components of a \keyword{tikzpicture} environment.
    This makes it possible to use these commands inside more complicated
    diagrams.

  \section{Young diagrams and tableaux}

    \emph{Partitions}, which are weakly decreasing sequences of
    non-negative integers, are fundamental objects in algebraic
    combinatorics and representation theory.\index{partition}  Rather
    than working with sequences of integers, it is often useful to
    visualise partitions as Young \emph{diagrams}, which are arrays of
    boxes in the plane. A \emph{tableau} is a (Young) diagram where the
    boxes are labelled using some alphabet. Equivalently, a diagram is a
    tableau with empty labels.
    \index{young diagram@Young diagram|see{diagram}}
    \index{diagram}
    \index{tableau}

    For example, $\lambda=(4,3,3,2,0,0,\dots)$ is a partition of $12$, since the
    entries sum to~$12$. It is customary to omit the zeros when writing partitions
    and to use exponents for repeated parts, so we also write
    $\lambda=(4,3^2,2)$. Many authors identity a
    partition~$\lambda=(\lambda_1,\lambda_2,\dots)$ with its
    \emph{diagram}, which is the set
    \[
    [\lambda] = \set{(r,c)|1\le c\le\lambda_r\text{ for }r\ge1}.
    \]
    The diagram of~$\lambda$ is visualised as a justified array
    of boxes in the plane, where the first row has~$4$ boxes, the nest
    two rows have~$3$ boxes and the last row has~$2$ boxes. The
    \emph{shape} of a diagram is the partition that gives the number of
    boxes in each row. Here are some examples of the diagrams and
    tableaux of shape $(2,1^2)$ that can be drawn using \aTableau:
    \begin{center}
      \tikzset{
         B/.style={fill=blue!20},
         G/.style={fill=ForestGreen!20},
         R/.style={fill=red!40,text=white},
         Y/.style={fill=yellow!20}
      }
      \Diagram{2,1^2}\qquad
      \Tableau{[R]1[B]2,[G]3,[Y]4} \qquad
      \Tableau[french]{[R]1[B]2,[G]3,[Y]4} \qquad
      \Tableau[ukrainian]{[R]1[B]2,[G]3,[Y]4} \qquad
      \Tableau[australian]{[R]1[B]2,[G]3,[Y]4}
    \end{center}
    This choice of colouring is gratuitous, but it emphasises that
    \aTableau allows you to add style (and, in particular,
    colour), to every box in these diagrams. This example also shows that
    \aTableau supports the four different conventions used for drawing Young
    diagrams and tableaux. The following table shows how to draw the
    Young diagram of shape $\lambda=(4,3^2,2)$ using the four tableaux
    conventions supported by \aTableau:
    \begin{center}
        \aTabSet{align=centre}
        \begin{tabular}{*3l} \toprule
            Convention & \aTableau command & Diagram \\ \midrule
            English    & \keyword{\Diagram[english]{4,3^2,2}}              & \Diagram{4,3^2,2}\\[10mm]
            French     & \keyword{\Diagram[french]{4,3,3,2}}      & \Diagram[french]{4,3^2,2}\\[10mm]
            Ukrainian\footnotemark
                       & \keyword{\Diagram[ukrainian]{4,3^2,2}}   & \Diagram[ukrainian]{4,3^2,2}\\[10mm]
            Australian\footnotemark
                      & \keyword{\Diagram[australian]{4,3,3,2}}  & \Diagram[australian]{4,3,3,2,0,0} \\[10mm]
            \bottomrule
        \end{tabular}
        \index{english@English}
        \index{french@French}
        \index{ukrainian@Ukrainian}
        \index{australian@Australian}
        \footnotetext[1]{Some authors call this the Russian convention}
        \footnotetext[2]{I have not seen this convention in the literature, but this name for the convention seems apt.}
    \end{center}
    As shown, partitions can either be typed out in full or using the,
    typically shorter, exponential notation. The different conventions
    for diagrams, and tableaux, are listed in order of their popularity
    in the literature.  By default, the \textit{English} convention is
    used, so \keyword{\Diagram{4,3,3,2}} produces the first diagram
    above.  These four conventions can be applied in the same way to the
    \keyword{\Tableau} command.  You can change the default convention
    using the \keyword{\aTabSet} command.

    This table shows how the commands provided by \aTableau work: there
    are basic commands for drawing diagrams, and these diagrams can be
    embellished using optional arguments and a key-value interface. In
    addition, as the following example shows, by supplying (optional)
    Cartesian coordinates, these diagrams can be used inside a
    \keyword{tikzpicture} environment to create more exotic pictures:

    \begin{Example}
      % \usetikzlibrary{patterns}
      \begin{tikzpicture}[
          B/.style={pattern=north east lines, pattern color=blue,opacity=0.5},
          Y/.style={pattern=north west lines, pattern color=ForestGreen,opacity=0.5},
        ]
        \Diagram(0,0)[ribbons={Y|16ccccrrr, B|16crrcccr}]{6,5^2,2}
        \draw[->](1.5, 1)node[right]{  $\alpha=(1,2)$} to [out=180,in=90]  (T-1-2.base);
        \draw[->](1.5,-2)node[right]{$f_\alpha=(4,2)$} to [out=180,in=270] (T-4-2.base);
        \draw[red,<->]([xshift=-1mm]T-1-1.south west)
           --node[rotate=90,above]{$l_\alpha=3$}([xshift=-1mm]T-4-1.south west);
      \end{tikzpicture}
    \end{Example}

    Most of the commands in this example are \TikZ commands, but most of
    the picture is drawn by the \keyword{\Diagram} command. The top of
    the example defines two \TikZ styles, \textsf{B} and \textsf{Y},
    which are used in the ribbons. The Young diagram is placed at
    position $(0,0)$ by the \keyword{\Diagram} command. The three
    \keyword{\draw} commands in this example are used to add the three
    arrows to the picture. You should consult the very readable \TikZ
    manual for an explanation of how these commands work.

    From the viewpoint of \aTableau, the most important point of this
    example is that the \keyword{\Diagram} command can be used inside a
    \keyword{tikzpicture} environment because it is equipped with the
    Cartesian coordinate $(0,0)$. The example also uses the four named
    coordinates, (\textsf{T-1-1}), (\textsf{T-1-2}), (\textsf{T-4-1}),
    and (\textsf{T-4-2}). When \aTableau draws a tableau, or diagram, it
    creates \TikZ named coordinates for each of the boxes in the
    diagram, so that (\textsf{T-r-c}) refers to the box in row \textsf{r} and
    column \textsf{c}. Giving finer control, the following standard
    \TikZ \textit{anchors} are available:

    \begin{Example}
      \begin{tikzpicture}[every node/.style={font=\tiny, text=gray}]
          \Diagram(0,0)[scale=4]{1}
          \node[label={above:(T-r-c)}] at (T-1-1){$\bullet$};
          \node[label={above:(T-r-c.north)}] at (T-1-1.north){$\bullet$};
          \node[label={below:(T-r-c.south)}] at (T-1-1.south){$\bullet$};
          \node[label={right:(T-r-c.east)}] at (T-1-1.east){$\bullet$};
          \node[label={left:(T-r-c.west)}] at (T-1-1.west){$\bullet$};
          \node[label={above right:(T-r-c.north east)}] at (T-1-1.north east){$\bullet$};
          \node[label={above left:(T-r-c.north west)}] at (T-1-1.north west){$\bullet$};
          \node[label={below right:(T-r-c.south east)}] at (T-1-1.south east){$\bullet$};
          \node[label={below left:(T-r-c.south west)}] at (T-1-1.south west){$\bullet$};
      \end{tikzpicture}
      \qquad
      \begin{tikzpicture}[every node/.style={font=\tiny, text=gray}]
          \Diagram(0,0)[ukrainian, scale=4]{1}
          \node[label={above:(T-r-c)}] at (T-1-1){$\bullet$};
          \node[label={above:(T-r-c.north)}] at (T-1-1.north){$\bullet$};
          \node[label={below:(T-r-c.south)}] at (T-1-1.south){$\bullet$};
          \node[label={right:(T-r-c.east)}] at (T-1-1.east){$\bullet$};
          \node[label={left:(T-r-c.west)}] at (T-1-1.west){$\bullet$};
          \node[label={above right:(T-r-c.north east)}] at (T-1-1.north east){$\bullet$};
          \node[label={above left:(T-r-c.north west)}] at (T-1-1.north west){$\bullet$};
          \node[label={below right:(T-r-c.south east)}] at (T-1-1.south east){$\bullet$};
          \node[label={below left:(T-r-c.south west)}] at (T-1-1.south west){$\bullet$};
      \end{tikzpicture}
    \end{Example}

    Of course, $r$ and $c$ shold be replaced by the row and column
    indices in the tableau --- and these anchors are only defined for
    the nodes in the tableau. These \textit{anchors} are a standard part
    of \TikZ. If you change the shape of the node then the available
    anchors may change, as described in the \TikZ manual.

    The previous example also shows that the \keyword{\Diagram} command
    accepts a \option{ribbons} option, which adds \textit{ribbons} to the
    diagram. Moreover, these ribbons can be given \TikZ styling. All of
    the options to these commands are described in the following sections.

    In the example above the Cartesian coordinate $(0,0)$ is necessary
    because the \keyword{\Diagram} command is embedded inside a
    \keyword{tikzpicture} environment, but the coordinate itself is not
    important in the sense that changing the coordinate will not change
    the picture. Cartesian coordinates become more meaningful when there
    are more components inside the \keyword{tikzpicture} environment.
    Consider:

    \begin{example}[lefthand width=0.14\textwidth, sidebyside gap=0mm,]
      % \usetikzlibrary{decorations.pathmorphing}
      \begin{tikzpicture}[wave/.style={thick,red,->,decorate,decoration=snake}]
        \aTabSet{ribbonstyle={fill=blue!20, opacity=0.4}}
        \Diagram(0,0)[e=2,show=residues,ribbons={13crc},prefix=A]{3,2}
        \Diagram(0,-2.5)[e=2,show=residues,ribbons={12rcr},prefix=B]{2^2,1}
        \draw[wave](0.5,-1.1)--(0.5, -2.4);
      \end{tikzpicture}
    \end{example}

    \subsection{Tableaux}\indexcmd{Tableau}
    This section describes the \keyword{\Tableau} command, which draws
    tableau or labelled diagrams. The syntax for this command is:

     \keyword{\Tableau (x,y) [options] {tableau entries}}

    \begin{syntax}
      \item[$(x,y)$]\index{xy@$(x,y)$} The Cartesian coordinates $(x,y)$
      are needed if and only if the tableau is inside a
      \keyword{tikzpicture} environment.
      \item[options] Optional arguments, described below.
      \item[tableau entries]\index{tableau entries} The
      \keyword{tableau entries} are given as a comma separated list, with commas
      separating rows and each \textit{token}, or braced-group of
      tokens, being in a separate column. As we explain below,
      optionally, each tableau entry can be prefixed by \TikZ styling
      commands.
    \end{syntax}

   We show by example how to use the \keyword{\Tableau} command, starting
   with basic usage and finishing with style. Inside \keyword{\Tableau}, the rows
   are separated by commas, with the column entries given by the
   ``letters'' in the entry for each row:

  \begin{example}
    \Tableau{ 1234, 56, 7, 8 }       \qquad
    \Tableau{ \alpha\beta\gamma\delta,
              xy\eta,
              \lambda\mu\nu,
              \pi\rho\sum }
  \end{example}

  You can put all of the tableau specifications on one line, without any
  spaces. Alternatively, as I do, you add spaces and line breaks between
  the tableau entries to improve readability (although, you cannot have
  blank lines inside a \keyword{\Tableau} command). As this example
  suggests, by default, the entries of a tableau are typeset as
  mathematics, but this can be changed.

  As the second example above shows, the tableau entries are
  \textit{not} just individual characters: they also include \TeX\
  commands. Tableaux frequently contain entries that are not single
  characters, or given by commands. Such entries can be inserted in a
  tableau by enclosing them in braces:

  \begin{example}
    \Tableau{ 13{10}{11}{2x}, 2{x^2} }
  \end{example}

  There is one additional caveat for braced-entries: \textit{any single
  braced entry in a row needs to be double-braced}. In practical terms,
  the need to double-bracing is shown by the following example:
  \index{tableau entries!double-braced}

  \begin{example}
    \Tableau{ 1345, 8{10}, {11}, {12} } \qquad
    \Tableau{ 1345, 8{{10}}, {{11}}, {{12}} }
  \end{example}

  The $\{11\}$ and $\{12\}$ in the first tableau are both treated as
  spanning two columns in the tableau because they are not
  doubled-braced; in contrast,they occupy only one column in the second
  tableau because they are double-braced.

  The need for double-bracing of singleton entries is a consequence of
  how \LaTeX3 detects braced commas in comma separated lists. If you
  want to have a comma as an entry in a tableau, then you should enclose
  it in braces.

  \begin{example}
    \Tableau{ 13{,}5, 8{10}, {{11}} }
  \end{example}
  \index{tableau entries!comma}

  As we have defined them, tableaux are always of \textit{partition}
  shape, in that the lengths of the rows form a weakly decreasing
  sequence. In fact, the \keyword{\Tableau} command also accepts
  tableaux of \textit{composition} shape, where the lengths of the rows
  are not necessarily in decreasing order. \index{composition}

  \begin{example}
    \Tableau{ 245,82751,231 }
  \end{example}

  Empty rows in tableaux and Young diagrams are not supported!

  \subsubsection{Adding style}
  \index{star}
  \index{*|see{star}}
  \index{tableau entries!style}
  \index{tableau entries!star}

  The \aTableau package provides several different ways to add \TikZ
  style commands to tableaux and diagrams. This is made possible by
  adding style prefixes to the tableau entries. To take advantage of
  these styles you will need at least rudimentary knowledge of how \TikZ
  styles work. At least to get started, the examples in this manual will
  help.

  The simplest way to change the style of a tableau entry is to give it
  a $*$-prefix in the tableau specifications. This is the easiest way to
  add some style to some of the nodes in the tableau:

  \begin{example}
    \Tableau{ 1*2*3*4*5, 6*789, {10}*{11}{12}, {{13}} }
  \end{example}

  By default, the $*$-entries are given the \TikZ styling
  \keyword{fill=blue!20}. You can change the default $*$-style using the
  \option{starstyle} key:


  \begin{example}
    \Tableau[starstyle={fill=red!20,draw=red,thick}]
            { 1*2*3*4*5, 6*789, {10}*{11}{12}, {{13}} }
  \end{example}

  Notice that the $*$-styling takes precedence over the default styling
  of the surrounding boxes. This is because the boxes with the default
  styling are placed first, in the order that they are entered, after
  which the boxes with styling are placed.

  The $*$-syntax is convenient if you want to emphasise some of the
  boxes in a tableau, using a single style. In fact, it is possible to
  give every box in the tableau its own style by putting the style
  specification inside square brackets, \keyword{[...]}, before the
  corresponding letter in the tableau specification.  You can mix the
  $*$-syntax and the $[....]$-style syntax for different entries,
  although you can only apply one of these style settings to any given
  node.

  \begin{example}
    \Tableau[french]{
        1[blue!20]2[circle]3[red]4[cyan]5,
        6*789,
        {10}*{11}{12},
        {{13}} }
  \end{example}

  This example shows that some precision is needed when changing the
  style because simply giving a colour will change both the colour used
  to fill the box and the text colour, which is why some of tableau
  entries above appear to be blank. Instead of using just a colour, use
  \keyword{text=...} to change the text colour, \keyword{fill=...} to
  change the fill colour, and \keyword{draw=...} to change the border
  colour. If you want to apply more than one style setting to any
  tableau entry, such as \keyword{circle,fill=orange}, then you need to
  enclose the all of style settings inside matching square and curly
  braces \keyword{[\{...\}]}.  This is necessary because commas are used
  to separate the rows of the tableau.

  \begin{example}
    \Tableau[australian]{
        1[fill=blue!20]2[{circle,fill=orange}]3[text=red]45,
        6*78[{draw=cyan,ultra thick}]9,
        {10}*{11}{12},
        [{dashed,draw=red,ultra thick}]{13} }
  \end{example}

  As this example shows, modifying the borders does not add very much
  emphasis because not very prominent. The orange circle looks too big,
  but it is what we asked for because it diameter matches the central
  width of the box. The next example shows that we get a better result
  for \textit{Australian} and \textit{Ukrainian} tableaux if we add a
  \textit{minimum size} specification.

  For complicated styles, or styles that are used often, we recommend
  defining the style outside of the \keyword{\Tableau} command by using
  the \keyword{\tikzset} command to define the style:

  \begin{example}
    \tikzset{
       B/.style={fill=blue!40, text=yellow},
       G/.style={fill=green!80!blue,circle,minimum size=5mm},
       R/.style={fill=red,text=white, font=\tiny},
       Y/.style={fill=yellow!20, text=blue}
    }
    \Tableau[french]{[R]1[B]2,[G]3,[Y]4} \quad
    \Tableau[ukrainian]{[R]1[B]2,[G]3,[Y]4} \newline
    \Tableau[english]{[R]1[B]2,[G]3,[Y]4} \quad
    \Tableau[australian]{[R]1[B]2,[G]3,[Y]4}
  \end{example}

  In this example, the \TikZ styles \textsf{B}, \textsf{G} \textsf{R},
  and \textsf{Y} are being applied to the tableaux entries labelled $1$,
  $2$, $3$, and $4$, respectively. The colours used here, and throughout
  this manual, are for demonstration purposes only: I recommend against
  such garish choices in any self-respecting document!

  \subsubsection{\textbackslash Tableau options}

  The optional $*$-styles and $[\ldots]$-styles are the main mechanism
  for styling individual nodes in a tableau. Using a key-value syntax,
  you can change other aspects of the tableaux produced by the
  \keyword{\Tableau} command.  The rest of this section describes these
  keys, their default values, and gives examples of how to use them.

  The options described below can be used with the \keyword{\Tableau}
  command by placing them as a comma separated list inside square
  brackets:
  \begin{center}
    \keyword{\Tableau[options]{tableau entries}]}\quad or \quad
    \keyword{\Tableau(x,y)[options]{tableau entries}]}.
  \end{center}
  The options can appear in any order, with later options having
  precedence over earlier ones.

  When used inside a \keyword{\Tableau} command, the options only affect
  that particular tableau. Most of these options can also be used in the
  \keyword{\aTabSet} command, to change the default settings of all
  following tableaux in the same \LaTeX\ group.  For example, if you put
  the command \keyword{\aTabSet[ukrainian]} in the preamble of your
  document then, by default, latter tableaux and Young diagrams will be
  drawn using the Ukrainian convention.

  \option[]{$(x,y)$}

  The $(x,y)$-coordinates are needed if, and only if,
  \keyword{\Tableau}command is used inside a \keyword{tikzpicture}
  environment, in which case $(x,y)$ gives the coordinates of the
  ``outside corner'' of the box in row~$1$ and column~$1$ of the
  tableau. The following example shows how the tableaux are placed using
  coordinates inside a \keyword{tikzpicture} environment. The example
  also shows that, by default, the tableau boxes are half a unit wide.

  \tikzset{
    addgrid/.code = {
      % draw coordinate axes
      \draw[help lines, step=0.5,thin, LightGrey](-0.5,-1.5) grid (4,2.5);
      \foreach \pt in {0,...,3} {
        \node[below,font=\tiny] at (\pt,-1.5){$\pt$};
      }
      \foreach \pt in {-1,...,2} {
        \node[left, font=\tiny] at (-0.5,\pt){$\pt$};
      },
    }
  }

  \begin{example}
    \begin{tikzpicture}[addgrid]
      \Tableau(0,0)    [english]   {123,45}
      \Tableau(0,0.5)  [french]    {123,45}
      \Tableau(2.5,0.5)[ukrainian] {123,45}
      \Tableau(2.5,0)[australian]{123,45}
    \end{tikzpicture}
  \end{example}

  \option[T]{prefix}[string/character]

  By default, the boxes can be referenced using the node names
  \textsf{(T-1-1)}, \textsf{(T-1-2)}, \textsf{(T-1-3)}, \dots, with
  \textsf{(T-$r$-$c$)} referring to the node in row~$r$ and column~$c$.

  \keyword{tikzpicture} environment, or if the \TikZ \textit{remember
  picture} style is being used, which the next example does.

  \newpage

    \Tableau[ukrainian, prefix=A, tikz={remember picture}]
      { 1[fill=blue!20]2[{circle,fill=orange}]3[text=red]45,

        [{dashed,draw=red,ultra thick}]{13}

  still reference the node names in last tableau. This allows us to do

  to your tableau. The syntax for adding ribbons is described in
  \autoref{S:RibbonTableaux}, which describes the
  \keyword{\RibbonTableau} command. The basic idea is that you specify
  the location of the head of the ribbon, which has minimum row index

  \subsection{Tabloids}
  \indexcmd{Tabloid}

  \typeout{TABLOIDS-----------------------------------------------}
  A \emph{tabloid} is an equivalence class of tableau where two tableaux
  are equivalent if they have the same entries in each row, up to a
  permutation. In the literature, tabloids are drawn with (outer wall)
  lines drawn above and below each row, and no side boundary walls. The
  \keyword{\Tabloid} command functions in the same way as the
  \keyword{\Tableau} command, except that it draws tabloids.

  \begin{example}
    \tikzset{R/.style={fill=red!20,circle}}
    \Tabloid[australian]{12[R]3,45}
    \Tabloid[english]{12[R]3,45}
    \Tabloid[french]{12[R]3,45}
    \Tabloid[ukrainian]{12[R]3,45}
  \end{example}

  \subsection{Ribbon tableaux}\label{S:RibbonTableaux}
  \indexcmd{RibbonTableau}

    A \emph{ribbon} in a tableau, or in a diagram, is a connected strip
    of nodes that does not contain a \Diagram[scale=0.3,outline=false,align=bottom]{2^2}-square.
    The \emph{head} of a ribbon~$R$ is the node $(r,c)\in R$ such that
    $r'>r$ and $c'<c$ whenever $(r',c')\in R$ and $(r',c')\ne(r,c)$.
    \begin{example}
      \RibbonTableau{11rrrrrr}
      \RibbonTableau{16ccccc}
      \RibbonTableau{1{10}crcrcrcrc}
    \end{example}
